# 营销智枢——数据资产与能力治理中心（MIH）

英文工程名：`marketing-intelligence-hub`。本项目是面向营业厅智脑 / 营销智脑的数据资产与能力治理中心前端产品化原型，用于内部评审、客户沟通、菜单确认、功能确认和端到端业务闭环演示。

## 技术栈

- Vue 3 + TypeScript + Vite
- Vue Router：集中路由与菜单配置
- Pinia：全局布局状态
- Element Plus：政企级管理后台组件
- ECharts：首页驾驶舱和健康度雷达图
- Axios：预留真实后端 HTTP 客户端
- SCSS：统一视觉风格
- Mock DB + Service：前端模拟真实领域对象、关系查询、业务动作、审计日志

## 启动方式

```bash
npm install
npm run dev
```

## 产品定位

MIH 用于统一治理政策文件、知识资产、规则、工具、技能、终端授权、审核发布和运行审计。当前阶段不连接真实后端，但前端已经按照生产系统的信息架构组织：

```text
政策文件 → 原始文件 → 智能编译 → Markdown → 知识切片 → 结构化规则
       → 技能包 → 审核发布 → 终端授权 → 智脑调用 Trace → 质量评估 → 学习整改
```

## 菜单说明

系统采用左侧一级/二级菜单 + 顶部栏 + 面包屑 + 内容区布局，一级菜单包括：

1. 工作台首页
2. 数据资产目录
3. 知识工程中心
4. 规则与技能中心
5. 工具中心
6. 评估与学习中心
7. 终端能力管理
8. 审核发布中心
9. 运行监控与审计
10. 系统管理

菜单与路由集中维护在 `src/router/menu.ts` 与 `src/router/index.ts`，后续新增页面只需补充菜单项和路由组件映射。

## Mock 数据与关联关系

`src/mock/mockDb.ts` 是前端内存版业务数据库，所有核心数据通过 ID 关联，不再只是孤立数组：

- 资产关联文件、元数据、版本、Markdown、切片、规则、技能、终端、Trace、审核和风险。
- 文件关联资产和编译任务。
- Markdown 关联来源文件、切片和规则。
- 切片关联资产、Markdown、规则、技能和 Trace。
- 规则关联资产、切片、技能和测试用例。
- 技能关联资产、规则、工具、终端、测试用例、审核和 Trace。
- Trace 关联会话、终端、技能、规则、知识切片、工具调用和回答质量评估。

兼容旧页面的数据导出仍保留在 `src/mock/data.ts`。

## 前端业务服务层

`src/services/domainService.ts` 模拟后端领域服务，页面通过 service/API 执行业务动作，而不是只在页面里改字段：

- `submitReview`：生成审核任务、改变对象状态、写入审计。
- `review`：通过 / 驳回 / 退回并联动对象状态。
- `publish`：生成发布记录、更新状态、写入审计。
- `rollback`：切换当前版本并更新资产状态。
- `startCompile`：模拟重新编译并刷新产物状态。
- `simulateTool`：模拟工具 Schema 校验、权限校验、调用结果和日志。
- `generateSkillDraft` / `saveSkillDraft`：自然语言生成技能草案并保存到技能库。
- `simulateBrainCall`：模拟终端智脑调用并生成 Trace / 会话记录。

`src/services/relations.ts` 提供聚合详情查询：

- `getAssetDetail(assetId)`
- `getSkillDetail(skillId)`
- `getTraceDetail(traceId)`
- `getReviewDetail(reviewId)`

## 推荐演示路径

### 1. 光伏并网政策治理闭环

```text
工作台首页
→ 资产总目录：打开“江西省分布式光伏并网政策”
→ 查看来源文件、Markdown、知识切片、规则、技能、终端授权、Trace 和健康度
→ 业务专家工作台：生成光伏并网导办技能草案
→ 技能库：查看技能依赖知识/规则/工具/终端
→ 审核发布中心：查看审核影响范围
→ 运行监控与审计：模拟“自然人申请光伏并网需要什么材料？”并查看 Trace
```

### 2. 居民阶梯电价解释闭环

```text
资产总目录：居民阶梯电价执行口径
→ 规则测试：输入居民电价事实 JSON
→ 工具测试：调用电价计算工具
→ 技能库：查看居民阶梯电价解释技能
→ Trace：查看自助终端调用链路和质量评估
```

### 3. 冲突治理与学习整改闭环

```text
知识冲突检测：查看光伏容量阈值冲突 / 历史补贴政策召回风险
→ 资产详情：查看冲突影响的切片和规则
→ 审核发布中心：确认变更影响
→ 回答质量评估 / 知识缺失池：创建整改任务
```

## 主要交互

- 菜单切换、面包屑展示。
- 表格搜索、筛选、新建、编辑。
- 资产 / 技能 / Trace 详情抽屉展示完整关联关系。
- 文件上传、预览、智能编译和影响范围。
- 提交审核、审核通过 / 驳回 / 退回、发布、回滚。
- 工具测试展示请求、响应、耗时、错误码和调用日志。
- 业务专家工作台生成技能草案，推荐政策、规则、工具和测试用例，并可保存到技能库。
- 终端智脑调用模拟，生成完整 Trace 或权限拦截 Trace。
- 首页指标、待办、风险、资产类型分布和调用趋势展示。

## 后续如何对接真实后端

1. 在 `src/api/http.ts` 中配置真实 `baseURL`、认证 Header、错误拦截与刷新 Token 策略。
2. 将 `src/api/index.ts` 中的 Mock service 调用替换为 Axios 请求，例如 `http.get('/assets/{id}/detail')`、`http.post('/review/submit')`。
3. 保持 `src/types` 中的数据模型与 Java DTO 对齐，必要时在 API 层做字段适配。
4. 真实后端建议提供聚合详情接口，避免页面自行拼装关系：资产详情、技能详情、Trace 详情、审核详情。
5. 后续可基于路由 meta、菜单配置和终端权限策略做动态菜单与操作权限控制。

